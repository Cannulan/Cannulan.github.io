## 为什么“接口混乱”会成为你的心头之患？

原因很简单——**大模型的 API 生态还没有统一**。

- **Grok**：xAI 出品的模型，接口格式和 OpenAI 类似，但细节上可能不同（比如模型名称、参数限制）。
- **OpenAI**：行业标杆，但模型太多（GPT-4o、o1、o3 系列），不同模型有不同的请求限制和收费方式。
- **Claude**：Anthropic 的模型，API 用的是 Amazon Bedrock 或自己的 Anthropic 格式，跟 OpenAI 完全不同。
- **Gemini**：Google 出品，有自己的 Gemini API 库，格式又自成一体。

你的代码要同时“伺候”这么多“主子”，不混乱才怪。尤其是在写 Node.js 示例时，调试过程最怕报“401 Unauthorized”或“400 Bad Request”，因为你根本分不清是模型的问题，还是自己写错了 base_url。
