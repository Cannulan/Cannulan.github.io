## 陷阱一：API调用里的隐形“模型升级”

这是最容易翻车的场景。

你在代码里设置的是 `gpt-3.5-turbo`，结果因为 OpenAI 暂停了对这个旧版模型在某些地区的支持，或者你的 API 请求参数写得不严谨，请求直接被系统“升级”到了 `gpt-4` 或 `gpt-4-turbo`。

**什么是“请求升级”？** 一句话解释：你点了一杯美式，店家直接给你上了杯拿铁，然后告诉你“美式没了，但你得付拿铁的钱”。

标准商用 API 中，如果一个模型请求出错，系统会自动按照“最优模型优先级”尝试，最后很可能落在一个更贵的模型上。而你的速度监控和成本告警如果没有设置好，等到看到账单时，数据已经被“升级”消费了很久。

## 陷阱二：“美元标价”和“人民币结算”的汇率游戏

很多海外 API 平台，官方给的是“美元/千Token”的价格，这个数字看起来很小。

但到了真正结算的时候，你会面对几个黑洞：

1. **兑换汇率：** 平台往往使用远高于市场汇率的“内部汇率”来折算人民币，差价可以去到 10%-20%。
2. **手续费：** 海外支付有 3%-5% 的通道费，这些费用通常不会在报价单上写出来，而是藏在最终的账单里。
3. **增值税和跨境税：** 如果你的公司需要正规发票，通过某些海外渠道开发票，会有额外税费。

所以，看上去 1 美元能买 1000 个 Tokens，最终你实际花费的人民币，比当天汇率算出来的要多得多。很多小团队把这个叫做“账单隐形膨胀”。
