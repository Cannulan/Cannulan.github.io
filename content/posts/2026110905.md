### 高级用法：批量调用降低成本

如果你想实现“1块钱调用1000次”的极致低成本，可以这样优化：

1. **启用流式输出**：通过`stream=True`让Token消耗更可控
2. **限制输出长度**：设置`max_tokens=200`，避免单次调用消耗过大
3. **使用缓存**：对高频重复问题，本地缓存Kimi的回答
4. **利用并发**：用`asyncio`实现并发请求，批量处理简单任务

以下是一个批量调用示例：

python
import openai
import asyncio

openai.api_base = "https://www.qianjuai.com/v1"
openai.api_key = "你的千聚aiAPI密钥"

async def call_kimi_async(prompt, model="kimi"):
    """异步调用Kimi，适合批量场景"""
    try:
        response = await openai.ChatCompletion.acreate(
            model=model,
            messages=[{"role": "user", "content": prompt}],
            temperature=0.7,
            max_tokens=200
        )
        return response.choices[0].message.content
    except Exception as e:
        return f"错误：{e}"

async def main():
    tasks = []
    prompts = [f"用一句话回答：数字{i}的平方是多少？" for i in range(20)]
    for prompt in prompts:
        tasks.append(call_kimi_async(prompt))
    results = await asyncio.gather(*tasks)
    for i, result in enumerate(results):
        print(f"问题{i+1}: {prompts[i]}")
        print(f"回答: {result}\n")

if __name__ == "__main__":
    asyncio.run(main())
