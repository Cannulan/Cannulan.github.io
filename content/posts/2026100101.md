from openai import OpenAI
import base64

    base_url="https://www.qianjuai.com/v1",
    api_key="你的千聚API密钥"
)

# 读取本地图片并转为base64编码

    base64_image = base64.b64encode(image_file.read()).decode('utf-8')

response = client.chat.completions.create(
    model="gemini-2.5-flash",
    messages=[
        {
            "role": "user",
            "content": [
                {"type": "text", "text": "请详细描述这张图片里有什么？"},
                {
                    "type": "image_url",
                    "image_url": {
                        "url": f"data:image/jpeg;base64,{base64_image}"
                    }
                }
            ]
        }
    ]
)

print(response.choices[0].message.content)

## 错误代码全解决：常见故障排查（新手必看）

新人第一次调用出现报错是很正常的事，不用慌张。我整理了千聚平台上调用 Gemini 2.5 Flash 最高频的 4 类错误及其终极解决方案。

**错误 1：`401 Unauthorized`（身份验证失败）**

- 表现：返回 `"error": {"code": "unauthorized", "message": "..."}`
- 原因：API Key 写错了或者压根没写。
- 解决方案：去千聚控制台重新复制粘贴 Key，检查 `api_key=` 字段里是否含有空格或多余的引号。建议直接打印出来看一看 `print(client.api_key)`。

**错误 2：`404` 或 `"Model not found"`（找不到模型）**

- 表现：返回 `"The model 'gemini-2.5-flash' does not exist"`
- 原因：可能是你在千聚平台上没有激活该模型的分组（默认分组只包含部分模型）。
- 解决方案：前往千聚控制台的“模型管理”中，查看你当前 API Key 绑定的分组是哪个。强烈建议默认使用“限时特价分组”或“默认（混合）”分组，这两个分组都原生支持 Gemini 2.5 Flash。

**错误 3：`429 Too Many Requests`（请求频率超限）**

- 表现：响应告诉你在忙，等会儿再试。
- 原因：短时间并发量过大，触发了平台的保护机制。
- 解决方案：在代码中增加 `time.sleep(1)` 或使用指数退避重试机制。其实千聚平台的并发上限很高，但如果你用的是免费额度，建议先小额慢速测试。

**错误 4：`413 Payload Too Large`（输入内容超长）**

- 表现：模型报错，无法处理你的请求。
- 原因：你发的内容超过了当前模型的 token 限制（比如默认是 1M，但你的文本超出了）。
- 解决方案：对于 Gemini 2.5 Flash，千聚已经预设了无限制的上下文扩展，但如果你的数据用力过猛（比如传了一个4小时的长视频），需要先对内容进行切分预处理。
