## 方案一：零依赖，纯HTTP接入（最省事）

如果你不想引入任何新的第三方库，只想用Java自带的工具类，最直接的方式就是用 `HttpURLConnection` 构造一个POST请求。千聚平台的接口**完全兼容OpenAI的请求格式**，这意味着你只需要把 `base_url` 设置成千聚的地址即可。

这是最贴近底层的实现方式，代码虽简单，但能让你完全理解整个调用流程。

java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.io.OutputStream;
import java.net.HttpURLConnection;
import java.net.URL;

public class QwenApiHttpExample {
    public static void main(String[] args) throws Exception {
        // 1. 设置[千聚ai聚合站](https://www.qianjuai.com/)的API端点，兼容OpenAI接口
        URL url = new URL("https://www.qianjuai.com/v1/chat/completions");
        HttpURLConnection conn = (HttpURLConnection) url.openConnection();

        conn.setRequestMethod("POST");
        conn.setRequestProperty("Content-Type", "application/json");
        // 2. 填入你在[千聚ai聚合站](https://www.qianjuai.com/)申请的API Key（请替换成你自己的）
        conn.setRequestProperty("Authorization", "Bearer sk-your-qianju-api-key");
        conn.setDoOutput(true);

        // 3. 构造请求体，这里我们指定调用 Qwen 模型
        String jsonInputString = "{\n" +
                "  \"model\": \"Qwen/Qwen2.5-72B-Instruct\",\n" +
                "  \"messages\": [{\"role\": \"user\", \"content\": \"请用Java写一个快速排序算法，只需要输出代码。\"}],\n" +
                "  \"max_tokens\": 512,\n" +
                "  \"temperature\": 0.7\n" +
                "}";

        // 4. 发送请求
        OutputStream os = conn.getOutputStream();
        byte[] input = jsonInputString.getBytes("utf-8");
        os.write(input, 0, input.length);
        os.flush();
        os.close();

        // 5. 读取响应
        BufferedReader br = new BufferedReader(new InputStreamReader(conn.getInputStream(), "utf-8"));
        StringBuilder response = new StringBuilder();
        String responseLine;
        while ((responseLine = br.readLine()) != null) {
            response.append(responseLine.trim());
        }
        br.close();

        // 6. 打印结果
        System.out.println("HTTP状态码: " + conn.getResponseCode());
        System.out.println("响应结果: " + response.toString());
    }
}


把代码替换好你的API Key后，直接运行main方法。如果一切顺利，你会在控制台看到一串JSON，里面就包含了Qwen写好的快速排序代码。核心逻辑非常清晰，没有一句废话。

## 方案二：使用官方SDK（更优雅，面向对象）

如果你更喜欢用官方SDK来做抽象管理，千聚平台同样提供了完美支持。你可以直接使用 `openai-java` 的官方客户端库，只需更改 `baseUrl`。

这种方式更适合大型项目集成，能让代码结构更清晰，也更方便做流式输出和错误处理。

**首先，在 `pom.xml` 中添加依赖：**

xml
<dependency>
    <groupId>com.theokanning.openai-gpt3-java</groupId>
    <artifactId>service</artifactId>
    <version>0.18.2</version>
</dependency>


**然后，编写调用代码：**

java
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.service.OpenAiService;
import java.time.Duration;
import java.util.Collections;
import java.util.List;

public class QwenApiSdkExample {
    public static void main(String[] args) {
        // 1. 创建一个OpenAiService实例
        //    指定[千聚ai聚合站](https://www.qianjuai.com/)的API地址和你的API Key
        OpenAiService service = new OpenAiService(
                "sk-your-qianju-api-key",
                Duration.ofSeconds(60)
        );
        service.setBaseUrl("https://www.qianjuai.com/v1");

        // 2. 构建消息列表
        List<ChatMessage> messages = Collections.singletonList(
                new ChatMessage("user", "请用Java写一个快速排序算法，只需要输出代码。")
        );

        // 3. 构建请求
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("Qwen/Qwen2.5-72B-Instruct")
                .messages(messages)
                .maxTokens(512)
                .temperature(0.7)
                .build();

        // 4. 发送请求并获取响应
        service.createChatCompletion(request)
                .getChoices()
                .forEach(choice -> {
                    System.out.println("Qwen 回复: ");
                    System.out.println(choice.getMessage().getContent());
                });

        // 5. 关闭服务
        service.shutdownExecutor();
    }
}


对比方案一的代码，你会发现使用SDK后，我们完全不需要手动处理JSON序列化、HTTP连接状态和异常捕获。SDK把这些都封装好了，代码更简洁，更接近业务逻辑本身。

| **优点**         | 零依赖、代码直观、理解底层通信     | 面向对象、代码简洁、流式交互友好 |
| **缺点**         | 需要手动处理JSON、需要自己写异常处理 | 需要引入第三方依赖               |
| **适用场景**     | 快速原型、项目无额外依赖限制       | 长期维护的大型项目、生产环境     |

## 性能与流式输出：让你的应用更智能

生产环境的API调用通常需要处理流式输出，也就是让大模型一个字一个字地“说出”它的推理过程，而不是等全部生成完才一次性返回。这样可以极大提升用户体验。

[千聚ai聚合站](https://www.qianjuai.com/)的接口原生支持 `stream: true` 参数。在HTTP方案中，你需要使用长连接来逐行读取数据流；在SDK方案中，只需要在构建 `ChatCompletionRequest` 时调用 `.stream(true)` 方法即可。

**SDK方案下的流式输出示例：**

java
import com.theokanning.openai.service.OpenAiService;
import io.reactivex.Flowable;

// (省略 创建service和构建request的步骤，同上)
// 将 createChatCompletion 替换为 streamChatCompletion
request.setStream(true); 
Flowable<ChatCompletionChunk> stream = service.streamChatCompletion(request);

stream.blockingForEach(chunk -> {
    // 处理每个chunk
    System.out.print(chunk.getChoices().get(0).getMessage().getContent());
});


这样做的好处是：用户在界面上可以实时看到AI的回复，而不是面对一个漫长的“等待加载中”的圈圈。在千聚平台下，流式输出的延迟非常低，无论是Qwen还是其他模型，体验都很流畅。

## 踩坑经验与最佳实践

在调试Java调用Qwen API时，有几个常见问题我帮你提前列出来，免得你走弯路。

**1. 请求超时问题**
大模型API响应时间通常较长，特别是处理复杂逻辑时。记得在代码里设置合理的超时时间。对于SDK方案，我们可以在构造`OpenAiService`时设置超时；对于HTTP方案，可以使用`conn.setConnectTimeout()`和`conn.setReadTimeout()`方法。千聚平台的稳定性很高（官方标称可用性99.9%），但网络波动仍存在，建议将读取超时设置为至少60秒。

**2. Token数量限制**
Qwen模型单次请求有最大Token限制（取决于模型大小）。如果你发送的提示词太长，或者生成的内容太多，会被截断。在请求体中设置合理的 `max_tokens` 参数，比如512或1024，避免生成不完整的内容。

**3. 模型名称的精确写法**
在千聚平台上，调用Qwen模型的名称不是简单的 `qwen`，而是遵循其模型命名规范，例如 `Qwen/Qwen2.5-72B-Instruct`。你可以在千聚后台的“模型列表”中查询到所有支持模型的精确名称。使用正确的模型名称是成功调用的第一步。

public static String translate(String text, String sourceLang, String targetLang) {

    ChatCompletionRequest request = ChatCompletionRequest.builder()
            .model("Qwen/Qwen2.5-72B-Instruct")

            .getChoices().get(0).getMessage().getContent();
