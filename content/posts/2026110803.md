## 避坑指南：90% 新手会犯的错

代码看起来简单，但我在实际使用中踩过不少坑。我把它们列出来，你一定要看：

### 坑1：没关代理或VPN就运行代码

这是最常见的错误。你用了千聚，就是为了不翻墙。但如果你本地还开着代理（比如Clash、V2ray），代理可能会干扰你的网络请求，导致连接失败。

**解方：** 运行代码前，请完全关闭任何代理或VPN软件，确保是纯粹的中国大陆网络环境。

### 坑2：模型名称写错了

千聚内部的模型映射方式和官方命名不太一样。你不能照着 Google 官方的模型名直接写。

**解方：** 去千聚的控制台查看 "可用模型" 列表，找到 Gemini 部分。正确的模型名通常是 `gemini-2.0-flash`、`gemini-2.0-pro`、`gemini-1.5-pro` 这种格式。千万不要加什么 `-latest` 或 `-001` 后缀。

### 坑3：API Key 没用对

千聚的 API Key 和你 OpenAI 的 Key 是两个系统。你在 OpenAI 里申请的 Key，在千聚的 `base_url` 下用不了。

**解方：** 确保你的 `api_key` 设置的是 **在千聚控制台创建的那个 key**，而不是别的平台的。

### 坑4：模型参数设置太激进

Gemini 模型对 `max_tokens`、`temperature` 等参数比较敏感。如果你像用 OpenAI 那样设置很大的 `max_tokens`（比如 4096 甚至 8192），可能会报错或无法生成。

**解方：** 刚开始用，先把 `max_tokens` 设置为 1024 或 2048，`temperature` 设置为 0.7。等能正常跑通了，再慢慢调整。

### 坑5：没有使用流式输出

非流式输出在调用大模型时，如果网络稍有波动，容易超时或断连。流式输出（`stream=True`）可以在一定程度上缓解这个问题，同时用户体验更好。

**解方：** 像我的示例代码一样，始终使用 `stream=True`。如果非要非流式，请设置较长的 `timeout` 参数（比如 60秒）。
