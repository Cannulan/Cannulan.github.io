### 雷区 05：流式输出和非流式输出搞混了通信参数

Qwen3-Max 模型在[千聚ai聚合站](https://www.qianjuai.com/)上支持流式和非流式两种模式，但如果你没在调用时显式声明 `stream=True` 或 `stream=False`（或者 undefined 时），平台默认的行为往往会让你措手不及。

**错误示范：**
- 调用非流式，期望一次性拿到全部回复 → 没传 `stream` 参数，但SDK默认可能是 `True` → 程序只拿到一个 `[DONE]` 标记。
- 调用流式，想实时输出文字 → 没传 `stream` 参数，结果等到所有回复出完了才一次性返回。

**正确做法：**
python
# ✅ 非流式：明确指定stream=False
response = client.chat.completions.create(
    model="qwen3-max",
    messages=[...],
    stream=False
)
print(response.choices[0].message.content)

# ✅ 流式：明确指定stream=True
response = client.chat.completions.create(
    model="qwen3-max",
    messages=[...],
    stream=True
)
for chunk in response:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")


这能保证不管千聚的默认设置如何，你的代码行为一定符合预期。
