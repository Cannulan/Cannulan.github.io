## 第二步：环境准备——避开依赖坑（0报错首关）

很多Java项目报错，都是因为依赖版本冲突。接入DeepSeek R1，其实你不用硬上什么专门的AI库，一个轻量级HTTP客户端足矣。

**推荐方案：**
用 `OkHttp` + `Fastjson` 或 `Jackson`。这是最灵活、冲突最少、调试最方便的组合。

**Maven 依赖示例：**

xml
<dependency>
    <groupId>com.squareup.okhttp3</groupId>
    <artifactId>okhttp</artifactId>
    <version>4.12.0</version>
</dependency>
<dependency>
    <groupId>com.google.code.gson</groupId>
    <artifactId>gson</artifactId>
    <version>2.10.1</version>
</dependency>


**避坑点：** 如果你项目里在用Spring Boot，且默认集成了 `Jackson`，就不要硬上 `Fastjson`，否则可能遇到序列化冲突。直接用 `Jackson`（即 `spring-boot-starter-web` 自带的那个）就足够了。

## 第四步：手写Java代码实现（手把手，零报错 Copy）

我们不依赖任何复杂框架，直接基于 `OkHttp` 写一个最纯净的调用类。这样能最大程度保证零报错，并且方便你移植到任何Java项目里。

**核心逻辑：**
1.  构建OpenAI兼容的请求体，携带模型名称和用户消息。
2.  设置 `Authorization` 头为你在千聚官网申请的Key。
3.  设置 `base_url` 为 `https://www.qianjuai.com/v1`。
4.  发送POST请求到 `/chat/completions` 端点。
5.  解析返回的JSON，提取“推理内容”（reasoning）和最终回答（content）。

**实战代码片段：**

java
import okhttp3.*;
import com.google.gson.*;

import java.io.IOException;

public class DeepSeekR1Client {

    // 核心配置 - 请替换为你自己在[[千聚ai官网](https://www.qianjuai.com/)](https://www.qianjuai.com/)申请的API Key
    private static final String API_KEY = "sk-your-key-here";
    private static final String API_BASE_URL = "https://www.qianjuai.com/v1";
    private static final String MODEL_NAME = "deepseek-reasoner";

    private final OkHttpClient client;
    private final Gson gson;

    public DeepSeekR1Client() {
        this.client = new OkHttpClient.Builder()
                .connectTimeout(30, java.util.concurrent.TimeUnit.SECONDS)
                .readTimeout(60, java.util.concurrent.TimeUnit.SECONDS)
                .build();
        this.gson = new Gson();
    }

    public String callR1(String userMessage) throws IOException {
        // 1. 构造请求体 (完全兼容OpenAI格式)
        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", MODEL_NAME);
        requestBody.addProperty("max_tokens", 4096); // R1的思维链较长，建议设置大一些

        JsonArray messages = new JsonArray();
        JsonObject systemMsg = new JsonObject();
        systemMsg.addProperty("role", "system");
        systemMsg.addProperty("content", "你是一个专业的Java技术顾问，请用中文回答。");
        messages.add(systemMsg);

        JsonObject userMsg = new JsonObject();
        userMsg.addProperty("role", "user");
        userMsg.addProperty("content", userMessage);
        messages.add(userMsg);

        requestBody.add("messages", messages);

        // 2. 构建请求
        Request request = new Request.Builder()
                .url(API_BASE_URL + "/chat/completions")
                .addHeader("Authorization", "Bearer " + API_KEY)
                .addHeader("Content-Type", "application/json")
                .post(RequestBody.create(gson.toJson(requestBody), MediaType.parse("application/json")))
                .build();

        // 3. 发送请求并获取响应
        try (Response response = client.newCall(request).execute()) {
            if (!response.isSuccessful()) {
                throw new IOException("API请求失败，错误码: " + response.code() + "，错误信息: " + response.body().string());
            }
            String responseBody = response.body().string();
            
            // 4. 解析响应，特别提取DeepSeek R1的“思考”内容
            JsonObject result = gson.fromJson(responseBody, JsonObject.class);
            JsonObject choice = result.getAsJsonArray("choices").get(0).getAsJsonObject();
            JsonObject message = choice.getAsJsonObject("message");

            // 关键！提取 reasoning_content 和 content
            String reasoning = "";
            if (message.has("reasoning_content")) {
                reasoning = message.get("reasoning_content").getAsString();
            }
            String finalAnswer = message.get("content").getAsString();
            
            // 返回结构化的结果
            return "【模型思考过程】\n" + reasoning + "\n\n【最终回答】\n" + finalAnswer;
        }
    }

    public static void main(String[] args) throws IOException {
        DeepSeekR1Client client = new DeepSeekR1Client();
        String response = client.callR1("请用Java写一个快速排序的示例，并解释时间复杂度。");
        System.out.println(response);
    }
}


**零报错要点强调：**
- **Key配置**：绝对不要在代码里写死Key，用一个配置文件或环境变量。`API_KEY` 变量是你注册[[千聚ai官网](https://www.qianjuai.com/)](https://www.qianjuai.com/)后，在后台“API密钥”里生成的。
- **超时时间**：DeepSeek R1思考链很长，默认的HTTP超时很容易翻车。以上代码将 `readTimeout` 设为了60秒，如果请求的问题超复杂，甚至可以再调大。
- **模型名称**：再次确认，是 `deepseek-reasoner`，不是其他。

## 第五步：常见报错场景（提前预防）

我帮你把90%的人都会遇到的坑，提前列出来。遇到了直接用以下方案秒解。

### 场景一：HTTP 401 Unauthorized
**原因**：API Key 不正确或未填写。
**解决方案**：确认你Key的格式是 `sk-` 开头的一串字符。如果从千聚后台复制，注意不要漏掉字符或者多复制了空格。

### 场景二：HTTP 400 Bad Request 或 404 Not Found
**原因**：模型名称 `model` 字段填错了。
**解决方案**：检查请求体中的 `model` 值是否是 `deepseek-reasoner`。千聚平台目前只支持这个ID。如果你私自填了 `deepseek-r1` 或者 `deepseek-chat`，系统不认识，就会直接报错。

### 场景三：返回文本超级长，程序卡死或报错
**原因**：R1的输出包括“推理内容”和“最终内容”，如果没做好长度控制，可能撑爆内存。
**解决方案**：代码中已经设置了 `max_tokens=4096`。如果你的场景要求更长，可以改成8192，但建议配合流式传输使用更稳妥。

### 场景四：无法解析 `reasoning_content` 字段
**原因**：代码里尝试获取 `reasoning_content`，但返回内容里可能没有这个字段（某些非R1模型就没有）。
**解决方案**：我们的代码已经用 `if (message.has("reasoning_content"))` 做了安全判断，不会因此NPE。这是正确的做法。
