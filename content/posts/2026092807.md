## 先搞懂 `base_url` 是什么

说白了，`base_url` 就是你程序里调用 AI 模型 API 时，请求发往的那个“根地址”。

比如你调用官方的 OpenAI API，代码里往往会写：

你的请求，无论是问 GPT-4 还是生成一张图片，都会先发到这个地址上。

对于 DeepSeek、Claude 这些模型，道理是一样的。它们都有自己独立的 `base_url`，比如 DeepSeek 的 `base_url` 是 `https://api.deepseek.com`，Claude 的则是 `https://api.anthropic.com/v1`。

这就产生了一个问题：**如果你要用三个不同模型，你得记住三个不同的 `base_url`，维护三套不同的 API 密钥，还得处理三个不同的计费体系。** 这已经够烦了，更别提有些平台你根本没法直接访问。
