#### 2. 核心代码：调用DeepSeek V3非流式接口

下面的示例是非流式调用（一次性返回完整回答），适合简单问答场景：

java
import okhttp3.*;
import com.google.gson.*;

import java.io.IOException;

public class DeepSeekV3Example {

    private static final String API_KEY = "你的千聚API密钥"; 
    private static final String BASE_URL = "https://www.qianjuai.com/v1"; 
    private static final String MODEL = "deepseek-v3"; // 模型名称

    public static void main(String[] args) throws IOException {
        OkHttpClient client = new OkHttpClient();

        // 构造请求体
        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", MODEL);
        requestBody.addProperty("max_tokens", 1024);

        JsonArray messages = new JsonArray();
        JsonObject userMessage = new JsonObject();
        userMessage.addProperty("role", "user");
        userMessage.addProperty("content", "用Java写一个冒泡排序算法");
        messages.add(userMessage);
        requestBody.add("messages", messages);

        // 构建HTTP请求
        Request request = new Request.Builder()
                .url(BASE_URL + "/chat/completions")
                .header("Authorization", "Bearer " + API_KEY)
                .header("Content-Type", "application/json")
                .post(RequestBody.create(MediaType.parse("application/json; charset=utf-8"), requestBody.toString()))
                .build();

        // 发送请求并处理响应
        try (Response response = client.newCall(request).execute()) {
            if (response.isSuccessful()) {
                String responseBody = response.body().string();
                JsonObject jsonObject = JsonParser.parseString(responseBody).getAsJsonObject();
                String content = jsonObject.getAsJsonArray("choices")
                        .get(0).getAsJsonObject()
                        .get("message").getAsJsonObject()
                        .get("content").getAsString();
                System.out.println("DeepSeek V3回答：\n" + content);
            } else {
                System.err.println("请求失败：" + response.code() + " - " + response.message());
            }
        }
    }
}


**关键点**：
- `BASE_URL` 直接填千聚平台的API地址，无需任何额外配置。
- `MODEL` 参数填 `deepseek-v3`（千聚平台上对应的模型标识）。
- 请求格式完全兼容OpenAI，`Authorization`用Bearer Token。

#### 3. 核心代码：调用DeepSeek V3流式接口

流式输出（SSE）更适合对话、代码生成等长文本场景——能让内容边生成边展示，用户体验更好。

java
import okhttp3.*;
import okhttp3.sse.*;
import com.google.gson.*;

public class DeepSeekV3StreamExample {

    private static final String API_KEY = "你的千聚API密钥";
    private static final String BASE_URL = "https://www.qianjuai.com/v1";

    public static void main(String[] args) {
        OkHttpClient client = new OkHttpClient.Builder()
                .readTimeout(30, java.util.concurrent.TimeUnit.SECONDS) // 流式调用需更长超时
                .build();

        // 构造请求体
        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", "deepseek-v3");
        requestBody.addProperty("max_tokens", 2048);
        requestBody.addProperty("stream", true); // 启用流式

        JsonArray messages = new JsonArray();
        JsonObject userMessage = new JsonObject();
        userMessage.addProperty("role", "user");
        userMessage.addProperty("content", "用Java写一个函数，计算斐波那契数列的前20项");
        messages.add(userMessage);
        requestBody.add("messages", messages);

        Request request = new Request.Builder()
                .url(BASE_URL + "/chat/completions")
                .header("Authorization", "Bearer " + API_KEY)
                .header("Content-Type", "application/json")
                .post(RequestBody.create(MediaType.parse("application/json; charset=utf-8"), requestBody.toString()))
                .build();

        // 创建SSE事件源
        EventSource eventSource = EventSources.createFactory(client)
                .newEventSource(request, new EventSourceListener() {
                    @Override
                    public void onEvent(EventSource eventSource, String id, String type, String data) {
                        if ("[DONE]".equals(data)) {
                            System.out.println("\n--- 流式输出结束 ---");
                            return;
                        }
                        try {
                            JsonObject jsonObject = JsonParser.parseString(data).getAsJsonObject();
                            String content = jsonObject.getAsJsonArray("choices")
                                    .get(0).getAsJsonObject()
                                    .get("delta").getAsJsonObject()
                                    .get("content").getAsString();
                            System.out.print(content); // 边接收边输出
                        } catch (Exception e) {
                            // 忽略解析错误
                        }
                    }

                    @Override
                    public void onFailure(EventSource eventSource, Throwable t, Response response) {
                        System.err.println("连接失败：" + t.getMessage());
                    }

                    @Override
                    public void onClosed(EventSource eventSource) {
                        System.out.println("连接关闭");
                    }
                });

        // 保持主线程存活一段时间等待完成
        try {
            Thread.sleep(30000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}


**流式调用注意**：
- 请求体要加上 `\"stream\": true`。
- 使用OkHttp的`EventSources`来处理SSE事件，每个`data`字段代表一段增量内容。
- 当收到 `[DONE]` 标记时，表示流式输出结束。

### 常见问题与避坑指南

#### 问题1：模型名填什么？

在千聚平台上，DeepSeek V3的模型标识是 `deepseek-v3`（注意大小写）。如果不确定，可以查看千聚平台的完整模型列表文档。

#### 问题2：接口返回超时怎么办？

DeepSeek V3推理速度很快，但如果你用的`max_tokens`很大，或者网络环境波动，建议在HTTP客户端设置合理超时。我的建议是：
- 非流式：`connectTimeout(10s)` + `readTimeout(30s)`
- 流式：`readTimeout(30s)` 或更长

#### 问题3：流式输出丢数据？

检查你的代码是否正确处理了SSE的`event`类型。千聚的流式接口完全兼容OpenAI格式，`data`字段的增量内容在`delta.content`中。

#### 问题4：“API Key无效”？

确保在千聚平台（www.qianjuai.com）上申请了API key，并且在HTTP请求头中正确设置：`Authorization: Bearer 你的key`。
