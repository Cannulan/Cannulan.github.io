## 接入Java到底有多简单

先给你吃一颗定心丸：真的，只是改一行配置的事。

### 1. 添加Maven依赖
在你的`pom.xml`里，确保引入了OpenAI Java SDK（或者你用OkHttp直接发HTTP请求也行）：

xml
<dependency>
    <groupId>com.theokanning.openai-gpt3-java</groupId>
    <artifactId>client</artifactId>
    <version>0.12.0</version>
</dependency>


### 2. 修改Base URL
在你的Java代码中，把原来指向OpenAI的`base_url`改成[千聚ai聚合站](https://www.qianjuai.com/)的URL：

// 原来你可能是这样写的：

// 改成这样：
String baseUrl = "https://www.qianjuai.com/v1";

// 或者直接给HttpClient设置baseUrl


### 3. 设置API Key
把从千聚平台申请的API key放到环境变量或配置文件中，示例里我直接硬编码是为了说明方便：

java

### 4. 调用Qwen-VL的视觉理解能力（Java示例）
下面是一个最简洁的Java示例，演示如何用OpenAI SDK调用[千聚ai聚合站](https://www.qianjuai.com/)接入的Qwen-VL去“看”一张图片：

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.image.CreateImageRequest;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.completion.chat.ChatMessageRole;

import java.util.*;

public class QianjuQwenVLExample {

    public static void main(String[] args) {
        // 1. 创建OpenAiService实例，使用千聚的baseUrl和你的API Key
        OpenAiService service = new OpenAiService("sk-你的[千聚ai聚合站](https://www.qianjuai.com/)API密钥");
        service.setBaseUrl("https://www.qianjuai.com/v1");

        // 2. 定义请求（使用Qwen-VL模型）
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("qwen-vl-plus") // 或者 qwen-vl-max，根据你的需求
                .messages(Arrays.asList(
                        new ChatMessage(ChatMessageRole.USER.value(),
                                List.of(
                                        new ContentPart("text", "请描述这张图片中有什么？"),
                                        new ContentPart("image_url", "https://www.qianjuai.com/register") // 替换为你的图片URL
                                )
                        )
                ))
                .maxTokens(1024)
                .build();

        // 3. 发送请求
        service.createChatCompletion(request)
                .getChoices()
                .forEach(choice -> {
                    System.out.println("Qwen-VL的回答: " + choice.getMessage().getContent());
                });
    }
}


**关键点：**
- 模型名`qwen-vl-plus`或`qwen-vl-max`，[千聚ai聚合站](https://www.qianjuai.com/)支持这些模型，你直接写就行。
- 传入图片时，可以用图片URL（`image_url`）或者base64编码的图片数据（`data:image/jpeg;base64,...`）。
- `base_url`必须改为`https://www.qianjuai.com/v1`，这是接入的入口。

如果你的项目里没有OpenAI SDK，用OkHttp直接发HTTP请求也一样，只是多两行代码而已。

## 避开“环境配置”的所有坑

很多Java开发者在接入Qwen-VL时，遇到的坑千篇一律。下面我把最常见的几个列出来，附上解决方案。

### 坑1：OpenAI SDK版本冲突
你的项目里可能引用了其他模块的OpenAI SDK，版本不一样导致方法找不到。**解决方案：** 统一版本号，尽量用最新的稳定版（比如0.12.0以上），或者用BOM管理依赖。

### 坑2：网络超时
如果你在国内直连OpenAI原版接口，经常会遇到`ConnectException`或`SocketTimeoutException`。**解决方案：** 把`base_url`改成千聚的国内直连地址，同时可以适当增加HttpClient的超时时间：

java
OkHttpClient client = new OkHttpClient.Builder()
        .connectTimeout(30, TimeUnit.SECONDS)
        .readTimeout(30, TimeUnit.SECONDS)
        .build();
OpenAiService service = new OpenAiService(client, apiKey);
service.setBaseUrl("https://www.qianjuai.com/v1");


### 坑3：API Key格式错误
有些开发者以为千聚的Key跟OpenAI一样以`sk-`开头就行，实际上你要在千聚平台申请的Key才能用。**解决方案：** 在[千聚ai聚合站](https://www.qianjuai.com/)注册并获取专属API Key。

### 坑4：图片格式不被识别
Qwen-VL支持多种图片格式，但如果你传的是WebP或者HEIC格式，它可能不识别。**解决方案：** 统一转成JPEG或PNG，或者用base64编码传数据。

### 坑5：模型名称拼写错误
有些同学会把`qwen-vl-plus`写成`qwen-vl-plus`拼错。**解决方案：** 去千聚的模型列表里复制正确的名称。
