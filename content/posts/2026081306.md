## Node.js调用方法二：原生HTTP请求，极致精简（适合极小项目）

如果你连`openai`包都不想装，想极致精简你的项目依赖，可以直接用Node.js内置的`fetch`或Node18+的`http`模块。这种方法代码量最少，但需要你手动处理一点JSON格式。

1.  **核心代码（用fetch）**：
    javascript
    async function callDoubaoDirect() {
      const response = await fetch('https://www.qianjuai.com/v1/chat/completions', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer 你的千聚API Key`,
        },
        body: JSON.stringify({
          model: 'doubao-pro-32k',
          messages: [{ role: 'user', content: '什么是[千聚api聚合站](https://www.qianjuai.com/)？' }],
        }),
      });
      const data = await response.json();
      console.log(data.choices[0].message.content);
    }

    callDoubaoDirect();
    

**为什么省钱？**
- **零依赖**：你的`package.json`里不会增加任何新依赖，打包体积小，启动速度快。
- **逻辑透明**：完全理解请求链路，适合初学者学习或调试。

## Node.js调用方法三：用Axios封装一个“省钱小工具”（适合批量处理）

如果你的项目里有大量的并发请求、重试逻辑，或者需要处理更复杂的错误，用`axios`封装一个专门的Doubao调用函数会更合适。

1.  **安装包**：`npm install axios`
2.  **封装工具函数**：
    javascript
    import axios from 'axios';

    const doubaoClient = axios.create({
      baseURL: 'https://www.qianjuai.com/v1',
      headers: {
        'Authorization': `Bearer 你的千聚API Key`,
        'Content-Type': 'application/json',
      },
    });

    export async function callDoubao(messages, model = 'doubao-pro-32k') {
      try {
        const response = await doubaoClient.post('/chat/completions', {
          model: model,
          messages: messages,
        });
        return response.data.choices[0].message.content;
      } catch (error) {
        // 处理错误，比如重试或降级
        console.error('调用Doubao失败:', error.response?.data || error.message);
        return null;
      }
    }
    

3.  **在项目中使用**：
    javascript
    const result = await callDoubao([{ role: 'user', content: '讲个冷笑话' }]);
    console.log(result);
    

**为什么省钱？**
- **复用性强**：封装好后，整个项目都能用，避免重复写请求代码。
- **错误处理**：可以精确控制超时、重试次数，避免因为网络抖动导致不必要的Token浪费。
